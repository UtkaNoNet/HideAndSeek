# HideAndSeek — плагин пряток для Minecraft (Paper 1.21.11)

## Как открыть в IDE

Это обычный Maven-проект, IDE сама подтянет зависимости из `pom.xml`.

**IntelliJ IDEA:**
1. File → Open → выбери папку `hideandseek` (ту, где лежит `pom.xml`).
2. IDEA сама распознает Maven-проект и предложит импортировать зависимости
   (или нажми на иконку Maven справа → "Reload All Maven Projects").
3. Дождись, пока подтянется `paper-api-1.21.11-R0.1-SNAPSHOT` — это может занять
   минуту-две при первом открытии (качается с `repo.papermc.io`).
4. Запусти конфигурацию Maven `clean package` (вкладка Maven → Lifecycle),
   либо `mvn clean package` в терминале IDE.

**Eclipse:** File → Import → Maven → Existing Maven Projects → укажи папку проекта.

**VS Code:** установи расширение "Extension Pack for Java" — оно само подхватит `pom.xml`.

Требования: **Java 21** (обязательно для Paper 1.21.11) и Maven 3.6+.

## Как собрать и запустить

1. `mvn clean package` в корне проекта.
2. Готовый jar — `target/HideAndSeek.jar`.
3. Кинь его в `plugins/` своего Paper 1.21.11 сервера (Java 21) и перезапусти сервер.

## Куда что класть / за что отвечает

Всё уже разложено по пакетам ровно так, как ты просил в архитектуре. Класть код
никуда не нужно — открой проект в IDE, собери (`mvn package`) и закинь jar на
сервер. Ниже — что где лежит и зачем.

```
com.hideandseek
│
├── Main.java            — точка входа плагина: onEnable/onDisable,
│                          регистрация команд и слушателей. Ничего игрового
│                          сам не делает, только "склеивает" остальные классы.
│
├── GameManager.java     — САМОЕ ГЛАВНОЕ. Машина состояний игры (GameState),
│                          весь игровой цикл: /start -> роли -> голосование ->
│                          прятки -> поиск -> победа -> сброс. Все решения о
│                          ходе игры принимаются тут, остальные классы —
│                          инструменты, которые GameManager вызывает.
│
├── GameState.java       — enum состояний (IDLE, ROLE_ASSIGN, VOTING, HIDING,
│                          SEARCHING, ENDING). Именно смена этого enum рождает
│                          строки "[DEBUG] GameState -> VOTING" и т.п.
│
├── commands/
│   ├── StartCommand      — /start, вызывает GameManager.startGame()
│   ├── SetHubCommand      — /sethub, сохраняет позицию хаба в config.yml
│   ├── SetMapCommand      — общий класс для /setforest /setdesert /setswamp
│   │                        (какая именно карта — передаётся при регистрации
│   │                        в Main.java, чтобы не плодить 3 одинаковых класса)
│   ├── StopCommand        — /stop, принудительно останавливает и сбрасывает игру
│   └── DebugCommand       — /debug on|off, включает [DEBUG]-сообщения
│
├── managers/
│   ├── VoteManager         — предметное голосование за карту (клик по предмету
│   │                         в инвентаре = голос), подсчёт и выбор победителя
│   ├── TimerManager        — универсальный обратный отсчёт с колбэками
│   │                         (используется и для 60 сек. пряток, и для 7 мин.
│   │                         игры, и для 20 сек. в конце)
│   ├── RoleManager         — случайный выбор искателя, выдача тайтлов,
│   │                         выдача лука/сердца по роли
│   ├── MapManager          — читает список настроенных карт из config.yml
│   ├── ScoreboardManager   — сайдбар с оставшимся временем и числом
│   │                         оставшихся прячущихся
│   └── VisibilityManager   — прячет подписи (nametag) над головами игроков,
│                             чтобы нельзя было опознать человека визуально
│
├── listeners/
│   ├── PlayerListener     — общие правила (отключение голода в игре),
│   │                        подстраховка на случай реальной смерти игрока
│   ├── DamageListener     — КЛЮЧЕВОЙ: превращает смертельный урон
│   │                        прячущемуся в "поимку" вместо реальной смерти
│   ├── InventoryListener  — обрабатывает клик по предмету голосования
│   ├── QuitListener       — правила выхода из игры (см. ТЗ: прячущийся
│   │                        вышел = погиб, искатель вышел = игра окончена)
│   └── JoinListener       — что происходит, если кто-то заходит на сервер
│                            посреди уже идущего раунда
│
├── model/
│   ├── GameMap      — id/имя/локация одной карты
│   ├── GameRole     — enum: NONE, SEEKER, HIDER, SPECTATOR
│   └── GamePlayer   — роль + флаг "жив/выбыл" конкретного игрока в раунде
│
└── utils/
    ├── ConfigUtil     — всё чтение/запись config.yml в одном месте
    ├── MessageUtil    — тайтлы, чат-сообщения, и debug-лог (виден только
    │                    админам с правом hideandseek.admin, только при /debug on)
    └── LocationUtil   — сборка Location из секции config.yml
```

## О чём стоит знать отдельно

- **Голосование за карту предметное**: игрокам выдаются предметы-иконки
  (саженец = лесной приют, песок = пепельные пески, кувшинка = топи),
  клик по предмету = голос. Логика замены иконок — в `VoteManager`.
- **Скрытие ников**: `VisibilityManager` убирает подпись над головой через
  scoreboard-команду (`NameTagVisibility.NEVER`). Это стандартными средствами
  Bukkit/Spigot. Если нужно ещё и подменить имя в tab-листе (полная анонимность
  вплоть до списка игроков) — это уже требует `ProtocolLib` и пакетной подмены,
  в этом каркасе не реализовано, но структура это не блокирует.
- **"Убийство" прячущегося на самом деле не убийство**: чтобы не было экрана
  смерти/респавна, `DamageListener` отменяет урон, если он смертельный, и сразу
  переводит игрока в `GameMode.SPECTATOR` через `GameManager.catchHider()`.
- **Равные шансы стать искателем**: `RoleManager.assignRoles()` каждый раз
  использует новый `Random` без сохранения истории между играми — то есть
  буквально "прошлые игры не учитываются", как ты и просил.
- **Debug Mode**: `/debug on` включает флаг в `config.yml` и в `MessageUtil`;
  дальше по всему коду разбросаны вызовы `MessageUtil.debug("...")` в местах
  смены состояния, победы в голосовании, назначения искателя, поимки игрока —
  именно они дают вывод вида из твоего примера.

## Что стоит донастроить самому

- Материалы предметов голосования (сейчас: саженец дуба / песок / кувшинка)
  — поменяй в `VoteManager.VOTE_ITEMS`, если хочешь другие иконки.
- Полная анонимизация ников в tab-листе — через ProtocolLib, если понадобится.
- `ChatColor` и `Player.sendTitle(...)` в `MessageUtil` рабочие, но помечены
  deprecated в пользу Adventure API (`net.kyori.adventure.text.Component` /
  `Audience.showTitle(Title)`). Для прод-плагина под 1.21.11 стоит со временем
  переписать `MessageUtil` на Adventure — структура это не блокирует, весь
  цветной текст идёт через один класс.
